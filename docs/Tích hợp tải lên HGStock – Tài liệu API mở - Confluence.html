Tài liệu về tích hợp tải lên HGStock – API mở

Ngày 08 tháng 01 năm 2026

Xác thực

Tất cả các request REST Admin API đều yêu cầu token hợp lệ để có thể truy xuất thông tin.

Sử dụng token trong header Authorization trong tất cả các API requests.

curl -X GET https://stock-api.hgmedia.app/demo-api \ -H 'Content-Type: application/json' \ -H 'Authorization: Bearer {token}'

Với quy trình xác thực khách hàng

curl -X POST {identity-server-url}/connect/token \ -H "Content-Type: application/x-www-form-urlencoded" \ -d "grant_type=client_credentials" \ -d "client_id={client_id}" \ -d "client_secret={client_secret}" \ -d "scope={scope}"

client_idvà client_secret, scopephải được admin HGStock cung cấp trước.

Với Implicit Flow

curl "{identity-server-url}/connect/authorize?client_id={client_id}&redirect_uri={redirect_uri}&response_type=token&scope={scope}"

client_secret không bắt buộc trong Implicit Flow.

Thông số chung:

identity-server-url: URL của Identity Server, thay đổi tùy theo môi trường sản phẩm.

Môi trường Development: https://accounts.hgmedia.dev

Môi trường Production: https://accounts.hgmedia.app

Lỗi

Mã lỗi

  • 400 Bad Request: data format không đúng định dạng.

  • 401 Unauthorized: client không có thông tin xác thực hợp lệ

  • 403 Forbidden: server từ chối phản hồi. Trường hợp chủ yếu là do ứng dụng sử dụng token vào các api không được cấp quyền.

  • 404 Not Found: tài nguyên được yêu cầu không tìm thấy, tuy nhiên có thể tìm thấy trong tương lai.

  • 429 Too Many Requests: client vượt quá ngưỡng giới hạn request

  • 5xx Errors: lỗi hệ thống nội bộ.

Định dạng dữ liệu

Ngày giờ :

Tuân thủ theo chuẩn ISO-8601 với timezone UTC.

Tất cả các dữ liệu datetime trong request và response của REST API đều tuân thủ theo quy chuẩn này

Ví dụ:

Lời yêu cầu

{ "resourceFile": { "createdDate": "2002-11-04T08:05:03Z" } }

Phản ứng

{ "resourceFile": { "updatedDate": "2002-11-04T08:05:03Z" } }

domain_url: URL của Stock BE, thay đổi tùy theo môi trường sản phẩm.

Môi trường Development: https://stock-api.hgmedia.dev

Môi trường Production: https://stock-api.hgmedia.app

Tiêu đề

X-Time-Zone: Timezone của client, ví dụ: Asia/Bangkok

Authenticantion: Mã thông báo người mang
Với client sử dụng Client Credentials Flow, userId được gửi qua query param, ví dụ:

curl -X GET https://stock-api.hgmedia.app/demo-api?userId=1123xxx \ -H 'Content-Type: application/json' \ -H 'Authorization: Bearer {token}'
